# cnweb-fnscore-archive

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态文件的形式保存，不针对任何特定的域名或网站。

## 项目简介

这是一个内容归档仓库，主要存放历史或特定用途的 HTML 页面副本。页面内容保持原始状态，仅作记录和查阅用途，不涉及任何动态服务或外部数据接口。

## 目录结构

```
cnweb-fnscore-archive/
├── README.md          # 本说明文件
├── pages/             # 存放归档的 HTML 页面文件
│   ├── page1.html
│   ├── page2.html
│   └── ...
└── assets/            # 页面引用的资源文件（如有）
    ├── images/
    ├── styles/
    └── scripts/
```

- `pages/`：主要归档目录，每个 HTML 文件为一个独立的页面存档。
- `assets/`：存放页面所需的辅助资源，如图片、CSS、JavaScript 文件等。

## 页面归档说明

- 所有归档页面均为静态 HTML 文件，不依赖外部服务器或 API。
- 页面内容可能包含样式、脚本或其他前端资源，这些资源已尽量内联或一并归档在 `assets/` 中。
- 归档时尽可能保持页面原始结构和功能，但部分交互（如需要后端支持的动态内容）可能无法完整呈现。
- 页面文件命名清晰，便于按需查阅。

## 维护说明

- 本仓库由维护者不定期更新，新增或补充归档页面。
- 欢迎提交 issue 或 pull request 来建议归档内容，但请确保提交的内容符合仓库定位（仅归档静态 HTML 页面，不涉及具体网站或域名）。
- 对于包含敏感或版权内容的页面，仓库将不予收录。
- 仓库内容仅供学习、存档或技术参考，请勿用于违规用途。

## 使用方式

1. 克隆仓库到本地：
   ```bash
   git clone https://github.com/your-username/cnweb-fnscore-archive.git
   ```
2. 直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件即可查看归档内容。
